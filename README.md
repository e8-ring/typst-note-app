# 仕様

## コア

- 最小単位 : ノート
  - これが 1 ファイルにあたる。
  - 1 画面（1 タブ）で表示されるのは 1 つのノート。
- ノートのもつ情報
  - 一意的な id（英数字 4 桁）= 実体ファイルの名前
  - ノートのタイトル（任意）
  - ユーザーが設定する複数のタグ
  - 作成日、最終更新日
    - 最終更新日は、勝手に設定するのではなく、更新日を更新するボタンを押すことで反映されるようにする？（誤操作防止のため）
    - デメリット : 更新ボタンの押し忘れ
  - 案 : 内容の種類（定義、主張、証明、例、…）
- ノートの管理方法について
  - 各ノートの実体ファイルの名前は id (英数字 4 桁)
  - 実体ファイルはアプリフォルダに階層なくそのまま格納する。
  - ノートの階層関係はアプリ上で別に仮想的に設定できるようにする。
  - 実体ファイルが削除されたら、仮想階層からも自動で削除する。
  - 案 : メモのグループ
    - 動機 : 1 つの文献を読むときのメモはひとまとめにしたい。
    - それ用のタグを作ってしまえばそれで済むかも。
- ノートのメタデータの保持方法について
  - 現在の案 :
    1. ノートの実体ファイルの 1 行目にすべて記述
    2. 別フォルダに同名のファイルを作りそこに記述
    3. すべてのノートのデータを記述する 1 つのファイルを作る
  - 検討 :
    - 1 つ目は、ノート本体が独立しないというデメリットがある。
    - 2 つ目は、サイズの小さいファイルを大量に作成することによる負荷が懸念される。
    - 以上を考えると 3 番目がよい？
  - 1 行に 1 ノートのメタデータを記述するという方式。1 行目の先頭に id を書く。
